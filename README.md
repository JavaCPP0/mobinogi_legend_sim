# 전설 패션 승급 시뮬레이터

게임 내 패션 승급(고급 → 레어 → 엘리트 → 에픽 → 전설) 과
패션 가챠(뽑기)를 함께 시뮬레이션하는 파이썬 프로젝트입니다.

머리 / 상의 / 하의 / 하의 / 장갑 4부위에 대해, 사용자가 입력한 초기 보유량과
가챠 사용 여부에 따라 다음을 여러 번 시뮬레이션해서 확인할 수 있습니다.

* 4부위 모두 전설을 완성할 수 있는지 여부(성공/실패)
* 각 회차별 총 사용 현금
* 각 부위별

  * 엘리트 → 에픽 승급 도전 횟수
  * 에픽 → 전설 승급 도전 횟수
  * 전설 완성 여부
* 가챠 모드일 경우

  * 각 회차별 가챠 뽑기 횟수
  * 부위별로 가챠에서 획득한 고급/레어/엘리트/에픽 개수
* 전체 요약 통계

  * 성공/실패 케이스별 현금 사용량 통계
  * 가챠 모드일 경우, 성공/실패 케이스별 가챠 뽑기 횟수 통계

## 프로젝트 구조

파일 구조는 다음과 같습니다.

```text
project_root/
  config.py        # 확률, 비용, 부위 이름 등 설정
  simulation.py    # 실제 시뮬레이션 로직 (입출력 없음)
  main.py          # 사용자 입력/출력, 시뮬레이션 반복 실행, 통계 출력
  README.md
```

각 역할은 다음과 같습니다.

* config.py

  * 승급 확률, 가챠 확률, 비용(현금), 부위 이름 등 상수 정의
* simulation.py

  * 한 번의 시뮬레이션을 수행하는 `simulate_one_run` 함수를 제공
  * 인자로 초기 상태, 가챠 사용 여부를 받아 결과 딕셔너리 반환
* main.py

  * 터미널에서 사용자에게 입력을 받고
  * `simulate_one_run`을 여러 회 실행하고
  * 각 회차별 상세 로그와 전체 요약 통계를 출력

## 승급·가챠 규칙

등급 구조는 다음과 같습니다.

* 고급 → 레어 → 엘리트 → 에픽 → 전설

부위는 네 가지입니다.

* 머리, 상의, 하의, 장갑
* 각 부위는 자신의 재료로만 승급할 수 있습니다.
  (예: 머리 엘리트로 상의 에픽을 만들 수 없음)

### 합성 / 승급 확률 및 비용

고급/레어 승급

* 고급 3개 → 레어 승급 도전

  * 성공 확률: 40%
  * 성공: 고급 3개 소모, 레어 1개 생성
  * 실패: 고급 2개만 소모, 고급 1개는 남음
  * 비용: 0원 (현금 소모 없음)
* 레어 3개 → 엘리트 승급 도전

  * 성공 확률: 30%
  * 성공: 레어 3개 소모, 엘리트 1개 생성
  * 실패: 레어 2개만 소모, 레어 1개는 남음
  * 비용: 0원 (현금 소모 없음)

엘리트/에픽 승급

* 엘리트 3개 → 에픽 승급 도전

  * 성공 확률: 25%
  * 성공: 엘리트 3개 소모, 에픽 1개 생성
  * 실패: 엘리트 2개만 소모, 엘리트 1개는 남음
  * 비용: 500원
* 에픽 3개 → 전설 승급 도전

  * 성공 확률: 20%
  * 성공: 에픽 3개 소모, 전설 1개 생성
  * 실패: 에픽 2개만 소모, 에픽 1개는 남음
  * 비용: 3,000원

공통 규칙 (실패 시 재료 한 개 남기기)

* 3개로 승급을 시도하면

  * 성공 시: 재료 3개 → 상위 1개
  * 실패 시: 재료 2개만 소모, 1개는 남음

### 가챠(패션 뽑기) 규칙

가챠 1회

* 비용: 2,000원
* 등급 확률

  * 에픽: 2%
  * 엘리트: 10%
  * 레어: 33%
  * 고급: 55%

어떤 부위에 나오는지

* 머리, 상의, 하의, 장갑 4부위에 균등 분배 (각 25%)
* 예: 가챠에서 엘리트가 나왔다면, 4부위 중 하나를 랜덤으로 골라 그 부위 엘리트 +1

## 시뮬레이션 모드

프로그램 실행 시, 먼저 가챠 사용 여부를 묻습니다.

```text
추가 과금 패션 뽑기를 사용하시겠습니까? (y/N):
```

* y / yes 입력
  → 가챠 모드 (부족한 재료는 가챠로 계속 보충)
* 그 외 / 그냥 Enter
  → 가챠 비사용 모드 (초기 보유 재료만 사용)

각 모드 동작은 다음과 같습니다.

가챠 비사용 모드 (use_gacha = False)

* 사용자가 입력한 에픽/엘리트 수량만 사용
* 에픽/엘리트가 부족해 더 이상 승급 시도를 할 수 없는데
  아직 4부위 전설이 아니라면 그 회차는 실패로 종료

가챠 모드 (use_gacha = True)

* 현재 재료로 더 이상 승급할 수 없으면

  * 패션 뽑기를 1회 실행하여 재료를 수급
  * 다시 승급 시도를 반복
* 4부위 모두 전설 1개씩 완성될 때까지 계속 진행
  (가챠 모드에서는 이론상 항상 성공으로 끝나지만, 비용이 크게 달라짐)

목표

* 항상 최종 목표는

  * 머리, 상의, 하의, 장갑 전설 1개씩 확보

## 사용 방법 (입력 흐름)

실행하면 차례대로 다음 입력을 받습니다.

1. 가챠 사용 여부

```text
추가 과금 패션 뽑기를 사용하시겠습니까? (y/N):
```

* y / yes → 가챠 모드
* 그 외 → 가챠 비사용 모드

2. 각 부위별 현재 보유 에픽/엘리트 개수

예시:

```text
머리 에픽 개수: 0
머리 엘리트 개수: 50
상의 에픽 개수: 0
상의 엘리트 개수: 40
하의 에픽 개수: 0
하의 엘리트 개수: 30
장갑 에픽 개수: 1
장갑 엘리트 개수: 40
```

고급/레어는 항상 0개에서 시작한다고 가정합니다.
(필요하면 나중에 입력 항목을 확장 가능)

3. 시뮬레이션 반복 횟수

```text
시뮬레이션 횟수 (기본값 30, 그냥 Enter):
```

* 그냥 Enter → 30회
* 숫자 입력 → 해당 횟수만큼 시뮬레이션 반복

## 출력 형식

각 회차별 상세 결과

예시:

```text
1회차:
머리: 에픽승급도전횟수:12, 전설승급도전횟수:2 성공
상의: 에픽승급도전횟수:8,  전설승급도전횟수:1 실패
하의: 에픽승급도전횟수:6,  전설승급도전횟수:0 실패
장갑: 에픽승급도전횟수:10, 전설승급도전횟수:1 성공
사용된 현금: 57,000원
가챠 뽑기 횟수: 15회
  머리 가챠 획득 → 고급:10, 레어:3, 엘리트:1, 에픽:0
  상의 가챠 획득 → 고급:8,  레어:2, 엘리트:0, 에픽:1
  하의 가챠 획득 → 고급:7,  레어:1, 엘리트:1, 에픽:0
  장갑 가챠 획득 → 고급:9,  레어:4, 엘리트:0, 에픽:0
```

설명

* 각 부위별

  * 에픽승급도전횟수: 엘리트 3개로 에픽을 시도한 횟수
  * 전설승급도전횟수: 에픽 3개로 전설을 시도한 횟수
  * 성공/실패: 최종적으로 그 부위가 전설 1개를 완성했는지 여부
* 사용된 현금: 각 회차에서 승급 + 가챠에 사용된 총 현금
* 가챠 모드일 때

  * 가챠 뽑기 횟수: 그 회차에서 가챠를 몇 번 돌렸는지
  * 각 부위 가챠 획득: 가챠 통해 얻은 고급/레어/엘리트/에픽 개수

가챠 비사용 모드에서는 가챠 관련 출력이 생략되고,
승급 비용만 포함한 사용 현금이 출력됩니다.

## 요약 통계 출력

모든 시뮬레이션이 끝나면 전체 요약을 출력합니다.

예시:

```text
===== 요약 =====
총 시뮬레이션 횟수: 30회
성공: 12회, 실패: 18회

[성공한 경우 기준 현금 사용량]
  평균: 85,300원
  최소: 62,000원
  최대: 120,000원

[실패한 경우 기준 현금 사용량]
  평균: 41,500원
  최소: 18,000원
  최대: 75,000원

[성공한 경우 기준 가챠 뽑기 횟수]
  평균: 23.4회
  최소: 15회
  최대: 40회

[실패한 경우 기준 가챠 뽑기 횟수]
  평균: 11.2회
  최소: 3회
  최대: 20회
```

설명

* 성공

  * 4부위 모두 전설 1개씩 완성된 회차 수
* 실패

  * 가챠 비사용 모드에서 재료가 바닥나 더 이상 승급 시도 불가한데
    전설 4부위를 달성하지 못한 회차 수
  * 가챠 모드에서는 일반적으로 실패가 거의 발생하지 않습니다.
* 성공/실패별 현금 사용량 통계

  * 평균 / 최소 / 최대
* 가챠 모드일 때만

  * 성공/실패별 가챠 뽑기 횟수 통계도 함께 출력

## 설정 변경 방법

게임 내 실제 값에 맞추고 싶다면 config.py를 수정하시면 됩니다.

예시:

```python
# config.py 일부

PROB_ELITE_TO_EPIC = 0.25
PROB_EPIC_TO_LEGEND = 0.20

COST_ELITE_TO_EPIC = 500
COST_EPIC_TO_LEGEND = 3000

COST_GACHA = 2000

GACHA_PROB_EPIC = 0.02
GACHA_PROB_ELITE = 0.10
GACHA_PROB_RARE = 0.33
GACHA_PROB_COMMON = 0.55
```

예를 들어

* 전설 도전 확률이 실제 게임에서 15%라면
  PROB_EPIC_TO_LEGEND = 0.15
* 가챠 1회 비용이 3,000원이라면
  COST_GACHA = 3000

처럼 변경한 뒤 다시 실행하시면 됩니다.

## 참고 사항

* 이 시뮬레이터는 난수(random)를 사용하기 때문에
  같은 입력, 같은 회수라도 실행할 때마다 결과가 달라질 수 있습니다.
* 실제 게임 체감과 비교하면서

  * 내 현재 재료/과금으로 전설 4부위까지 갈 확률
  * 대략 어느 구간의 금액대에서 전설 4부위를 기대할 수 있는지
    를 감 잡는 용도로 사용하시기 좋습니다.
